# ascent-domgen

Pure-Scala, JVM-only generator that emits ascent's typed DOM / CSS / ARIA catalogs from a
**pinned, network-free** vendored W3C [webref](https://github.com/w3c/webref) snapshot. It is a
build-time tool only — **never a runtime dependency** of any ascent module.

## Regenerating the catalogs

```bash
sbt "domgen/runMain ascent.domgen.Main"
```

Run this after editing a generator template or re-vendoring a newer webref snapshot, then review
the generated diff and commit it. There is no network access — the generator reads only the
committed snapshot, so output changes only when the generator or the vendored data changes.

> `sbt domgen/run` won't work: the module has several `ZIOAppDefault` entry points (the inspectors
> below), so the main class must be named explicitly with `runMain`.

## Do not hand-edit generated files

Every emitted file starts with:

```
// AUTO-GENERATED by ascent-domgen. Do not edit by hand — see domgen/README.md to regenerate.
```

To change generated output, edit the generator (this module) or the vendored input under
[`data/`](../data/), then regenerate. Hand-edits are overwritten on the next run.

## Inputs (vendored, pinned)

- [`data/webref/`](../data/webref/) — the W3C webref snapshot; the exact upstream commit is in
  [`data/webref/PINNED_COMMIT.txt`](../data/webref/PINNED_COMMIT.txt). Provides HTML elements
  (`elements/html.json`, plus vendored SVG specs), DOM events (`events.json`), the IDL
  (`idlparsed/`), and the CSS specs (`css/*.json`).
- [`data/aria-query/`](../data/aria-query/) — `ariaPropsMap.js`, the ARIA property catalog.

## Outputs

Written into the `generated/` source trees of the consuming modules:

| Target | Files |
|--------|-------|
| `dom-types/.../generated/` | `Elements`, `Attrs`, `Events`, `Enums`, `EnumAccessors`, `AriaAttrs` |
| `dom-facade/.../generated/` | `Interfaces`, `Facades`, `Dictionaries`, `EnumAccessors` (js-only DOM facade) |
| `dom-core/.../generated/`  | `Elements`, `ElementsMemory`, `ElementFactory` |
| `js/.../generated/`        | `TypedEvents` (typed event-handler DSL) |
| `css/.../generated/`       | `StylesGenerated`, `StylesValueTraits`, `MediaFeatures`, `ContainerFeatures`, `FontFaceDescriptors`, `PageDescriptors`, `CounterStyleDescriptors`, `PseudoSelectors` |

## Inspecting the snapshot

Two read-only inspectors dump what the snapshot contains, for debugging a grammar or attribute:

```bash
sbt 'domgen/runMain ascent.domgen.AttrInspect [interface] [attr]'
sbt 'domgen/runMain ascent.domgen.css.CssInspect <propertyName>'   # or: selectors | atrules
```
